# Автоматическое формирование отчетности

Веб-приложение на Flask + MySQL для создания, редактирования, предпросмотра и экспорта отчетов и шаблонов.

## Стек

- Python Flask
- MySQL / XAMPP
- HTML, CSS, JavaScript
- SQLAlchemy
- pytest

## Быстрый запуск

1. Создайте виртуальное окружение:
   ```powershell
   py -m venv venv
   .\venv\Scripts\Activate.ps1
   ```

2. Установите зависимости:
   ```powershell
   py -m pip install -r requirements.txt
   ```

3. Создайте локальный `.env` из примера:
   ```powershell
   Copy-Item .env.example .env
   ```

4. Заполните параметры базы данных в `.env`.

5. Запустите MySQL в XAMPP и создайте базу:
   ```sql
   CREATE DATABASE ft_reports_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
   ```

6. Инициализируйте таблицы:
   ```powershell
   py -m flask --app app.py init-db
   ```

7. При необходимости заполните локальную базу демонстрационными данными FT NET:
   ```powershell
   py -m flask --app app.py seed-ftnet-demo
   ```

8. Запустите приложение:
   ```powershell
   py app.py
   ```

По умолчанию приложение доступно на `http://127.0.0.1:5000`.

## Демонстрационные данные FT NET

Для локальной демонстрации ВКР можно подготовить реалистичное наполнение организации FT NET:

```powershell
py -m flask --app app.py init-db
py -m flask --app app.py seed-ftnet-demo
py app.py
```

Команда `seed-ftnet-demo` идемпотентна: повторный запуск обновляет демонстрационные записи и не создает дубликаты организации, сотрудников, групп, папок, шаблонов, отчетов и связей доступа.

Для уже существующей учетной записи Геннадия Кудрявцева пароль не меняется. Для новых демонстрационных пользователей временный пароль берется из переменной окружения `DEMO_USER_PASSWORD`. Если переменная не задана, команда генерирует случайный временный пароль и выводит его в консоль только во время выполнения seed-команды.

Демонстрационные данные предназначены для локального запуска. В production-окружении seed требует явного разрешения через `ALLOW_DEMO_SEED=true` или параметр `--force-production`.
